**Description:** An interactive sign language translator designed to empower hearing-impaired and aphonic individuals by using advanced AI to instantly translate sign language gestures into readable/audible output in real time. This is a comprehensive capstone project with full ML pipeline, human-centered design research, and production deployment architecture.

**Language Breakdown:**

|----------|-----------|

**Directory Structure:**

```
├── client/               # Frontend application
├── server/               # Backend infrastructure

├── data/                 # Raw, processed, labels
├── notebooks/            # 4 Jupyter workflows
│   ├── 1_data_exploration

│   ├── 3_episode_evaluation
│   └── 4_user_usability_demo
├── SETUP.md
├── docker-compose.yml
└── requirements.txt
```

**ML Architecture — Dual-Model System:**

| Model | Architecture | Input | Latency Target |
|-------|-------------|-------|----------------|
| CNN Frame-Wise Classifier | Conv → BatchNorm → ReLU → MaxPool (×3) → Dense → Softmax | 21 hand landmarks × 3 coords per frame | <100 ms/frame |
| LSTM Sequence Classifier | TimeDistributed Dense → 2 LSTM layers → Dropout → Dense → Softmax | 15–30 consecutive frames | ~150 ms |

**Performance Targets:**
- Sub-150 ms latency end-to-end
- ≥95% accuracy across varied lighting and hand orientations
- ≥80 System Usability Scale (SUS) score

**Data Pipeline:**
- Input: MP4/AVI videos at 30 FPS, minimum 640×480
- Augmentations: Rotation (±10°), scaling (±10%), Gaussian noise (σ=0.01)
- Train/Val/Test: 80/10/10 split with reproducible seeds

**Key Dependencies:** MediaPipe Hands, TensorFlow, PyTorch, OpenCV, FastAPI, Uvicorn, Dovetail SDK

**Human-Centered Design:**

Three target user personas: Deaf/Hard-of-Hearing individuals, Educators/Caregivers, and AR/VR enthusiasts. Research conducted across four stages: contextual interviews and affinity mapping (Dovetail), prototype testing with 10 participants, longitudinal remote studies with 20 testers (Qualtrics), and real-time refinement with latency/accuracy benchmarking.

**Deployment Options:**
- Real-time webcam inference: `python src/infer.py --model_path checkpoints/cnn_best.h5 --use_webcam True`
- Batch video processing: `python src/infer.py --model_path checkpoints/lstm_best.h5 --input_videos data/raw/videos/ --batch_mode True`
- FastAPI REST endpoint accepting base64-encoded frames

**Future Roadmap:** Full ASL alphabet coverage, TensorFlow Lite/PyTorch Mobile for smartphones, depth sensor fusion, saliency heatmaps for explainability, cross-cultural validation, industrial IoT integration

**Stats:** 58 commits (most active repo)

### 4.12 Robust-Fake-Profile-Detection-Using-Deep-Neural-Networks-with-Adversarial-Training

> **URL:** [github.com/yeoleshweta/Robust-Fake-Profile-Detection-Using-Deep-Neural-Networks-with-Adversarial-Training](https://github.com/yeoleshweta/Robust-Fake-Profile-Detection-Using-Deep-Neural-Networks-with-Adversarial-Training)

**Description:** Detects fraudulent Instagram accounts using deep neural networks enhanced with adversarial robustness techniques. Compares standard DNN training against adversarial-augmented training.

**Language:** Jupyter Notebook (100%)

**Files:**
```
├── Original_FinalProject_FakeProfileDetection_DNN.ipynb
├── README.md
└── Robust Fake Profile Detection Using DNNs with Adversarial Training.pdf
```

**Dataset:** 236 Instagram profiles (28 genuine, 208 fake — highly imbalanced). Features include follower counts, username length, privacy status, and business classification. SMOTE applied for class balancing; StandardScaler normalization.

**DNN Architecture:**
- Input → 64 neurons → 32 neurons (ReLU, 0.2 dropout) → 32 neurons (ReLU, 0.3 dropout) → 2 neurons (softmax)
- Optimizer: Adam (lr=0.001), Loss: Weighted cross-entropy, Epochs: 200

**Adversarial Attack Methods Evaluated:**

| Attack | Type |
|--------|------|
| FGSM | Fast Gradient Sign Method |
| PGD | Projected Gradient Descent |
| CW | Carlini–Wagner |

**Results:**

| Model | Scenario | Accuracy | Real Precision/Recall | Fake Precision/Recall |
|-------|----------|----------|----------------------|----------------------|
| Standard DNN | Clean data | 92% | 0.65 / 0.79 | 0.97 / 0.94 |
| Standard DNN | FGSM/PGD attack | 88% | 0.00 / 0.00 | 0.88 / 1.00 |
| Standard DNN | CW attack | 83% | 0.00 / 0.00 | 0.88 / 0.94 |
| Robust DNN | Clean data | 91% | 0.57 / 0.82 | 0.97 / 0.92 |
| Robust DNN | All attacks | 88% | 0.49–0.50 / 0.82 | 0.97 / 0.88–0.89 |

**Key Finding:** The adversarially-trained model maintains consistent detection across all attack scenarios, preventing complete failure on the minority (genuine) class.
